# Event Validation & Processing - Comprehensive Fix Checklist

## ✅ PROBLEM IDENTIFICATION

**Root Cause**: Events silently dropped due to validation failures
- Messages polled successfully from Kafka ✓
- **BUT**: Validation fails silently, events are dropped without logging details
- Results in DB tables showing NULL for key fields (invoices, payments show ❌, customers ✓)

**Why This Happens**:
```python
# BaseAgent._validate_event() - OLD
except ValidationError as e:
    logger.error(f"Event validation failed: {e}")  # ❌ Minimal logging
    return None  # ❌ Silent drop

# _handle_message() - OLD
if event is None:
    return  # ❌ No indication message was dropped
```

---

## ✅ FIXES APPLIED

### Fix #1: Enhanced Validation Logging in BaseAgent
**File**: `agents/base/base_agent.py:369-387`

**Before**:
```python
except ValidationError as e:
    logger.error(f"Event validation failed: {e}")
    return None
```

**After**:
```python
except ValidationError as e:
    # DIAGNOSTIC: Log the raw data that failed validation
    logger.error(f"❌ EVENT VALIDATION FAILED (Agent: {self.agent_name})")
    logger.error(f"   Raw data: {raw_data}")
    logger.error(f"   Validation errors: {e.errors()}")

    # Log missing fields specifically
    raw_keys = set(raw_data.keys()) if isinstance(raw_data, dict) else set()
    required_fields = {'event_id', 'event_type', 'event_source', 'event_time', 'entity_id', 'payload'}
    missing = required_fields - raw_keys
    if missing:
        logger.error(f"   Missing required fields: {missing}")

    return None
```

**Impact**:
- You'll now see EXACTLY what data failed validation
- Missing fields are explicitly listed
- Can identify pattern of failures

### Fix #2: Drop Event Notification in _handle_message
**File**: `agents/base/base_agent.py:316-320`

**Before**:
```python
if event is None:
    return
```

**After**:
```python
if event is None:
    # DIAGNOSTIC: Message was dropped due to validation failure
    logger.warning(f"[{self.agent_name}] Dropping message from {topic}:{getattr(message, 'partition', '?')}:{getattr(message, 'offset', '?')} - validation failed")
    return
```

**Impact**:
- You'll see: `[DBAgent] Dropping message from acis.invoices:0:1234 - validation failed`
- Can track how many messages are being dropped per topic
